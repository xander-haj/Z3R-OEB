# Dev Tools

Standalone tools in this folder read generated dev-tool inputs.

## Overworld Viewer

Run this command from the project root:

```bash
python3 assets/restool.py --dump-overworld
```

Then serve the project root:

```bash
python3 -m http.server 8000
```

Then open:

```text
http://127.0.0.1:8000/dev_tools/overworld_viewer/
```

If you get this error:

```text
OSError: [Errno 48] Address already in use
```

that means something is already using port `8000`. Use another port:

```bash
python3 -m http.server 8001
```

Then open:

```text
http://127.0.0.1:8001/dev_tools/overworld_viewer/
```

Any unused port works. The port in the command and the port in the URL must
match.

The command must be run from the project root, not from `dev_tools/`, because
the viewer reads `assets/overworld_dump/` by relative path.

If the viewer reports that the overworld dump is stale, run the dump command
again. The viewer requires the sprite CHR and sprite palette files generated by
the current dump format.

## Overworld Workbench

For the full editor and mod pipeline reference, see:

```text
dev_tools/OVERWORLD_WORKBENCH.md
```

Run this command from the project root after extracting resources and generating
the base overworld dump:

```bash
python3 dev_tools/overworld_editor/server.py
```

Then open the URL printed by the server.

The Workbench creates data-only mod packages under:

```text
mods/overworld/<mod_id>/
```

Build or validate those packages from the project root:

```bash
python3 assets/modtool.py list
python3 assets/modtool.py validate mods/overworld/<mod_id>
python3 assets/modtool.py build mods/overworld/<mod_id>
```

Generated local assets are written under:

```text
assets/generated/overworld_mods/<mod_id>/
```

To compile or dump with a generated mod state, use:

```bash
python3 assets/restool.py --apply-overworld-mods <mod_id>
python3 assets/restool.py --dump-overworld --mod <mod_id>
```
